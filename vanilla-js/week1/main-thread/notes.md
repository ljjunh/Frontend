# 콜 스택(Call stack, 호출 스택)

자바스크립트는 단일 프로그래밍 언어로, 한 번에 하나의 작업만 처리
콜 스택은 실행 중인 코드의 위치를 추적하는 데이터 구조

"자바스크립트는 슈퍼맨 한 명이 모든 일을 하는 것과 같다."

- 호출 스택(Call Stack)이 하나만 존재
- 동시에 여러 작업을 병렬로 처리할 수 없음
- 함수 호출 시 스택이 쌓임(push)
- 함수 실행 완료 시 스택에서 제거됨(pop)
- LIFO(Last In, First Out) 원칙을 따름(후입선출)

블로킹 연산 주의

- 오래 걸리는 동기적 작업은 전체 실행을 차단
- 예: 무한 루프, 큰 파일의 동기적 읽기 등

스택 오버 플로우

- 엔진이 허용하는 스택보다 더 많은 스택이 쌓여 초과된 상태

스레드 안정성

- 경쟁 조건이나 데드락 같은 문제 발생 X
- 공유 자원 접근에 대한 복잡성 감소
