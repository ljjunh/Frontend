1. JS는 싱글스레드라 경쟁조건이 없다고 했는데, 비동기 상황에서도 경쟁조건이 발생 안함?
2. 데드락은 JS에서 완전히 불가능한건가?
3. 경쟁조건/데드락을 피하기 위해 JS/React 코드에서 어떤 패턴을 쓰나?

# 경쟁 조건

- 여러 스레드/프로세스가 공유 자원에 동시에 접근하려 할 때, 실행 순서에 따라 결과가 달라지는 현상

# 데드락

- 두 개 이상의 스레드/프로세스가 서로의 자원을 기다리며 영원히 블록되는 상태
- 4가지 필요 조건: 상호 배제, 점유 대기, 비선점, 순환 대기

1. 상호 배제(Mutual Exclusion)

- 자원은 동시에 하나의 스레드(또는 워커)만 사용할 수 있어야 한다

2. 점유와 대기(Hold and Wait)

- 자원을 점유한 채 다른 자원을 기다리는 상황

3. 비선점(No Preemption)

- 점유한 자원을 강제로 뺏을 수 없는 상태

4. 순환 대기(Circular Wait)

- A가 B를 기다리고, B가 A를 기다리는 등 순환적인 대기 상태

실제 생활 비유: 지하철역에서 만나기로 한 두 사람

- A와 B는 "지하철역 앞에서 만나자"고 약속
- 둘 다 "상대방이 먼저 도착하면 연락하겠지"라고 생각
- 그래서 아무도 연락을 안함
- 둘 다 도착했지만, 서로를 배려해서 연락을 안 하고 계속 기다림

1. 상호 배제 : 연락은 한 명이 먼저 해야 함. 동시에 할 수 없다고 생각함
2. 점유와 대기 : 둘 다 "나는 도착했고, 너의 연락을 기다리는 중"
3. 비선점 : 상대방에게 전화를 걸기보다는 받기를 기다림. "기다리는 중"이라는 입장 유지
4. 순환 대기 : A는 B의 연락을 기다리고, B는 A의 연락을 기다림 - 서로 연락을 안 함

문제점

- 시스템이 정지된 것처럼 보임
- 외부 개입 없이 해결되지 않음

데드락 방지

- 자원 획득 순서 통일
- 타임아웃 설정
- 가능하면 동시성 감소
